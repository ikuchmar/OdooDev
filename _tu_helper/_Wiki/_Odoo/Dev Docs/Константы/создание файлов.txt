При создании параметров в модели res.config.settings в Odoo следует учитывать следующие правила:

Укажите поле типа, которое соответствует типу параметра, который вы хотите создать. Например, для текстового параметра используйте fields.Char,
для числового параметра - fields.Float или fields.Integer, для булевого параметра - fields.Boolean и т.д.

Установите атрибут config_parameter=True для поля параметра. Это указывает, что это поле будет использоваться в качестве параметра настроек.

Определите атрибут default, чтобы задать значение по умолчанию для параметра.
Например: default=lambda self: self._default_default_value(), где _default_default_value() - это метод, возвращающий значение по умолчанию.

Если требуется ограничить допустимые значения параметра, вы можете использовать атрибуты selection или domain.
Например, selection=[('option1', 'Option 1'), ('option2', 'Option 2')] задает список допустимых значений для параметра.

Если параметр должен быть видимым только для определенных групп пользователей, вы можете указать атрибут groups.
Например, groups="base.group_user" ограничивает доступ к параметру только для пользователей, принадлежащих группе "base.group_user".

Пример создания параметра в модели res.config.settings:

from odoo import fields, models

class ResConfigSettings(models.TransientModel):
    _inherit = 'res.config.settings'

    my_parameter = fields.Char(string='My Parameter', config_parameter=True, default='Default Value')
В данном примере создается текстовый параметр my_parameter с начальным значением "Default Value".

============================================================================
set_values() и get_values()
============================================================================

после создания параметра в модели res.config.settings необходимо реализовать методы set_values() и get_values(), которые будут обрабатывать сохранение и получение значений параметров.


============================================================================
XML и PY файлы, связанные с моделью res.config.settings, обычно размещаются в следующих папках в структуре модуля:
============================================================================
но я делаю отдельную папку config_settings
-----------------------------------------


Для XML-файлов: my_module/views/, где my_module - имя вашего модуля. В этой папке обычно размещаются файлы с представлениями форм и виджетами для настроек.

Для PY-файлов: my_module/models/, где my_module - имя вашего модуля. В этой папке обычно размещаются файлы с определением модели res.config.settings и других связанных классов или методов.
